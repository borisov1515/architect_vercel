## Часть 1. Бизнес-описание

### 1.1. Цель продукта

Создать интерактивную учебную систему, которая помогает **понять** интеграционную архитектуру Salesforce, а не заучить ответы на практические тесты. Пользователь должен уметь:

- проходить вопросы в разных режимах (как на экзамене и в обучении);
- видеть **почему** ответ верный и почему остальные неверны;
- изучать материал **по темам экзамена** (секции blueprint);
- читать теорию, шпаргалки и глоссарий отдельно от практики;
- отслеживать прогресс и слабые места.

Продукт работает **в браузере**, доступен **локально** и **онлайн** (бесплатный хостинг).

### 1.2. Целевая аудитория

- Основной пользователь: кандидат на экзамен Integration Architect (self-study).
- Вторично: повторное прохождение после неудачной попытки, refresh знаний.

### 1.3. Источники контента

| Источник | Что даёт | Роль в продукте |
|----------|----------|-----------------|
| Set-1 … Set-4 (HTML, `pages/`, PDF) | 4 × 65 вопросов, **отдельные сеты** | 4 независимых банка; пользователь выбирает сет |
| Ручное / AI-наполнение | Правильные ответы, объяснения, теги тем | **Нет готовых ключей** — заполняем `explanation` по ходу |
| Авторский контент | Теория, выжимки, шпаргалки паттернов | Раздел «Теория» |

**Важно:** исходники в `Latest Exam/` не меняются. Продукт в `result/` импортирует их в единую структуру данных.

### 1.4. Ключевые пользовательские сценарии

1. **«Хочу пройти как на экзамене»** — 65 вопросов, таймер, без подсказок, итоговый балл.
2. **«Хочу разобрать тему»** — выбираю секцию blueprint → читаю теорию → решаю вопросы только по этой теме.
3. **«Угадываю, не понимаю»** — после ответа читаю развёрнутое объяснение; вопрос попадает в «повторить позже».
4. **«Хочу быстро повторить»** — flashcard или daily quiz на 10 вопросов.
5. **«Готовлюсь неделю»** — dashboard показывает прогресс по темам, streak, слабые зоны.

### 1.5. Критерии успеха (бизнес)

| Метрика | Цель |
|---------|------|
| Покрытие вопросами | Все 4 сета импортированы (4 × 65, отдельно) |
| Покрытие теорией | 6 секций blueprint + глоссарий + шпаргалка паттернов |
| Объяснения | 100% вопросов с полем explanation (поэтапное наполнение) |
| Удобство | Прохождение экзамена за одну сессию без перезагрузки |
| Доступность | Работает офлайн (PWA) и онлайн на бесплатном хостинге |

### 1.6. Ограничения и допущения

- **MVP без бэкенда:** прогресс, закладки, заметки — в `localStorage` браузера.
- **Без мультипользовательской авторизации** на первых этапах (один пользователь = один браузер).
- Правильные ответы и объяснения — **готовых ключей нет**; поле `explanation` заполняется поэтапно (AI + ручная правка).
- Язык интерфейса: **русский** (вопросы экзамена — **английский**, как в оригинале).

### 1.7. Решение по фреймворку и хостингу (бизнес-язык)

**Фреймворк:** **Vite + React + TypeScript**

**Почему:**
- Современный стандарт для интерактивных веб-приложений.
- Быстрая разработка UI с множеством режимов (квиз, теория, статистика).
- Один код — локально (`npm run dev`) и онлайн (статический деплой).
- Легко подключить PWA, тёмную тему, роутинг между разделами.

**Хостинг:** **Vercel** — [vercel.com](https://vercel.com)

Именно **Vercel** — бесплатный хостинг для React/Vite: подключаешь GitHub, каждый push → живой URL + preview. Hobby tier = $0.

**Альтернативы (равнозначные по цене = $0):**

| Платформа | Когда выбрать |
|-----------|---------------|
| **Vercel** | Основной выбор; лучший DX для React |
| **Netlify** | Аналог Vercel, чуть другой UI |
| **Cloudflare Pages** | Если нужен максимально быстрый CDN |
| **GitHub Pages** | Минимализм, только статика, без serverless |

**Не выбираем сейчас:** Next.js — избыточен, пока нет серверных API и SEO-требований. Vanilla JS — быстро превратится в неподдерживаемую кашу при 30 фичах.

### 2.4. Модель данных

#### Question (вопрос)

```typescript
interface Question {
  id: string;                    // "set3-q12" — уникальный
  setId: 1 | 2 | 3 | 4;         // обязательно; сеты не смешиваются
  setQuestionNumber: number;     // 1..65 внутри сета
  sectionId: string;             // "integration-fundamentals"
  tags: string[];                // ["platform-events", "async"]
  difficulty?: "easy" | "medium" | "hard";
  type: "single" | "multi";      // один или несколько ответов
  stem: string;                  // основной текст (HTML или markdown)
  context?: { type: "ul" | "ol"; items: string[] };
  diagramNote?: string;          // описание диаграммы
  options: {
    id: "A" | "B" | "C" | "D" | "E";  // оригинальная буква в PDF
    text: string;
  }[];
  correctAnswer: ("A" | "B" | "C")[];  // ["B"] или ["A","C"]
  explanation: {
    summary: string;             // краткий вывод
    correct: string;             // почему верный
    incorrect: Record<string, string>;  // A: почему нет, B: ...
    examTip?: string;            // что запомнить на экзамен
    relatedTheoryIds?: string[];
  };
  hasDiagram: boolean;
}
```

### 2.5. Секции экзамена (тегирование)

По официальному **Integration Architect Exam Outline**:

| ID | Секция (EN) | Секция (RU) | Ориентир веса |
|----|-------------|-------------|---------------|
| `fundamentals` | Integration Fundamentals | Основы интеграции | ~23% |
| `architecture` | Integration Architecture | Архитектура интеграции | ~20% |
| `api-design` | API Management & Design | API и дизайн интеграций | ~18% |
| `security` | Security & Identity | Безопасность и идентификация | ~17% |
| `data` | Data Integration | Интеграция данных | ~12% |
| `operations` | Testing, Monitoring & Operations | Тестирование, мониторинг, эксплуатация | ~10% |

Тегирование — полуавтоматическое (ключевые слова) + ручная правка при наполнении explanation.

### 2.6. QuizEngine — ядро логики

Отвечает за:

1. **Формирование сессии** — список questionId по режиму.
2. **Shuffle вопросов** — Fisher–Yates.
3. **Shuffle опций** — перемешивание с сохранением маппинга `displayIndex → originalId` для проверки ответа.
4. **Проверка ответа** — сравнение выбранных originalId с `correctAnswer`.
5. **Scoring** — подсчёт для exam mode.
6. **Фильтры** — по section, set, bookmarks, wrong-only, SRS due.

## Часть 3. Детальное описание функций (30 фич)

### Блок A. Режимы прохождения (фичи 1–3, 7, 14, 17, 27)

#### Фича 1. Прохождение подряд (Sequential)

| | |
|---|---|
| **Бизнес** | Пользователь проходит вопросы в фиксированном порядке, как в PDF/HTML сета. |
| **Техника** | Режим `mode: "sequential"`. Порядок = `setQuestionNumber` или глобальный индекс. Навигация: «Далее» / «Назад» (в учебном режиме). В exam mode — только вперёд. |
| **UI** | Индикатор «Вопрос 12 из 65», progress bar. |
| **Настройки** | Выбор сета (1–4) или «все сеты подряд». |

#### Фича 2. Вопросы вперемешку (Shuffle Questions)

| | |
|---|---|
| **Бизнес** | Исключает запоминание порядка; приближает к реальному экзамену. |
| **Техника** | `shuffleQuestions(ids, seed?)`. Опциональный seed для воспроизводимости сессии. |
| **UI** | Переключатель в настройках сессии перед стартом. |

#### Фича 3. Случайный порядок вариантов ответа

| | |
|---|---|
| **Бизнес** | Нельзя запомнить «всегда B» — каждый раз опции в новом порядке. |
| **Техника** | При рендере вопроса: `shuffledOptions = shuffle(options)`. Хранить `optionMapping: { displayKey → originalId }`. При submit сравнивать originalId. |
| **UI** | Метки A/B/C на экране — **позиционные** (первая, вторая, третья), не привязаны к PDF-букве. |

#### Фича 7. Режим «Экзамен» (Timed Exam)

| | |
|---|---|
| **Бизнес** | Симуляция реального экзамена: 65 вопросов, ~105 минут, без объяснений до финиша. |
| **Техника** | `mode: "exam"`. Timer countdown в store. Блокировка explanation до `session.complete`. Pass threshold: ~68% (44/65) — настраиваемо. |
| **UI** | Таймер в header, предупреждение за 10 мин, экран результатов: score, pass/fail, разбивка по секциям. |
| **Данные** | Сохранение `ExamSession` в localStorage для истории. |

#### Фича 14. Flashcard-режим

| | |
|---|---|
| **Бизнес** | Быстрый прогон: только вопрос → «показать ответ» → «знал / не знал». |
| **Техника** | `mode: "flashcard"`. Без выбора опций — показ correct + summary. Кнопки «Знал» / «Не знал» влияют на SRS. |
| **UI** | Карточка с flip-анимацией. |

#### Фича 17. Multi-select вопросы

| | |
|---|---|
| **Бизнес** | Часть вопросов экзамена: «choose two» — нужны чекбоксы, не radio. |
| **Техника** | `question.type === "multi"`. Submit активен когда выбрано N ответов. Scoring: all-or-nothing или partial (настраиваемо; по умолчанию all-or-nothing как на SF exam). |
| **UI** | Чекбоксы вместо radio. Подсказка «Select 2 answers». |

#### Фича 27. Режим «Без оценки» (Study / Tutor)

| | |
|---|---|
| **Бизнес** | Учёба без стресса: сразу после выбора — объяснение, без влияния на exam score. |
| **Техника** | `mode: "study"`. `showExplanationImmediately: true`. Попытки пишутся в stats, но не в exam history. |
| **UI** | Бейдж «Учебный режим». |

## Часть 5. Деплой на Vercel

### Локальная разработка

```bash
cd result/
npm install
npm run dev          # http://localhost:5173
npm run build        # dist/
npm run preview      # проверка production build
```

### Онлайн (Vercel)

1. Репозиторий на GitHub (папка `result/` или monorepo).
2. [vercel.com](https://vercel.com) → Import Project → выбрать repo.
3. Framework Preset: **Vite**.
4. Build Command: `npm run build`, Output: `dist`.
5. Каждый `git push` → автодеплой + preview URL.

**Стоимость:** $0 на Hobby tier (достаточно для личного use).

### Альтернатива: Netlify

Аналогично: `netlify deploy --prod` или connect Git. Build = `npm run build`, publish = `dist`.
