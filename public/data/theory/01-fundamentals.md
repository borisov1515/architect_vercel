# Основы интеграции

## Паттерны (запомни логику, не ответы)

| Паттерн | Когда | Пример |
|---------|-------|--------|
| **Request and Reply** | Нужен ответ сразу в UI | Checkout, credit check, bank account ID |
| **Fire and Forget** | Достаточно отправить, ответ не ждём | Уведомление внешней системы |
| **Remote Call-In** | Внешняя система пишет в SF | Webhook → REST API |
| **Batch** | Большие объёмы, не real-time | Nightly ETL, 2M records |

## Вопросы на экзамене

- Синхронный UI + долгий внешний сервис → **Continuation**, не блокирующий callout
- Несколько систем с разными форматами → **middleware / ESB**
- Минимум кода в SF → **Flow, Outbound Messaging, CDC** вместо Apex где можно
